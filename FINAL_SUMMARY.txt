HOW IT WORKS
================================================================================

WORKFLOW:
  1. User selects SOURCE FOLDERS (multiple allowed)
  2. User selects DESTINATION FOLDER
  3. User chooses ACTION (MOVE/COPY/NOTHING)
  4. App SCANS all media files recursively
  5. App EXTRACTS metadata from each file
  6. App CREATES CSV database with all metadata
  7. App SEPARATES files without GPS (if MOVE/COPY selected)
  8. App SHOWS dialog: "Start AutoTag?"
  9. User CONFIRMS and sets time window
  10. App MATCHES timestamps within window
  11. App WRITES GPS to matched files
  12. DONE!

OUTPUT STRUCTURE:
  destination/
  ├── database_20260306_1530.csv     ← All metadata
  └── NOGPS_20260306_1530/           ← Files without GPS
      ├── IMG_001.jpg                ← Gets GPS via AutoTag
      ├── IMG_002.jpg                ← Gets GPS via AutoTag
      └── VID_010.mp4                ← Gets GPS via AutoTag

================================================================================
KEY FEATURES
================================================================================

METADATA EXTRACTION:
  • Photos: EXIF data via AndroidX ExifInterface
  • Videos: QuickTime atoms via MP4Parser
  • Fields: GPS, timestamps, camera info, file details
  • Safety: Read-only during database build

GPS WRITING:
  • Only modifies: GPSLatitude, GPSLongitude, GPSAltitude
  • Preserves: Orientation, timestamps, camera data
  • Safe: No corruption or data loss
  • Format: Standard EXIF for photos

AUTOTAG ALGORITHM:
  • Reads file timestamp
  • Searches database for GPS-tagged files
  • Filters by time window (±5 to ±60 minutes)
  • Selects closest timestamp match
  • Copies GPS coordinates

TIME WINDOWS AVAILABLE:
  • ±5 minutes
  • ±10 minutes
  • ±15 minutes (default)
  • ±30 minutes
  • ±60 minutes
